【pkg 卡在 0% 或闪退】

原因：pkg 要从 GitHub 下载 Node 二进制，国内网络经常很慢或超时，窗口就可能卡住或直接关掉。

您可以选下面任一方式：

----------------------------------------
方式一：用「便携版」（推荐，不依赖 exe）
----------------------------------------
1. 打开本目录下的「客户包_免exe」文件夹。
2. 把里面的内容（整个文件夹或打成 zip）发给客户。
3. 客户电脑需要已安装 Node.js（从 https://nodejs.org 下载安装）。
4. 客户双击「启动龙虾客户端.bat」即可运行，效果和 exe 一样。

这样不需要等 pkg 下载成功，也能立刻交付。

----------------------------------------
方式二：开代理后再打包 exe
----------------------------------------
若您有 VPN 或代理：
1. 打开系统代理或 VPN，确保能访问 GitHub。
2. 在 PowerShell 里执行（把 127.0.0.1:7890 改成您的代理地址和端口）：
   set HTTPS_PROXY=http://127.0.0.1:7890
   set HTTP_PROXY=http://127.0.0.1:7890
   cd 项目根目录
   npx pkg scripts/vip-build/vip-lobster-entry.cjs --targets node18-win-x64 --output dist/vip-lobster.exe
3. 打包成功后，把 dist\vip-lobster.exe 和 .env.vip 复制到「交付给客户」文件夹。

----------------------------------------
方式三：手动下载 pkg 缓存（高级）
----------------------------------------
pkg 的 Node 二进制可从以下地址手动下载后放进缓存目录，再重新运行「生成exe.bat」：
https://github.com/vercel/pkg-fetch/releases

缓存目录：项目根目录下的 pkg-cache（生成exe.bat 已会自动使用）。
具体文件名可查 pkg 文档或报错里的路径。
