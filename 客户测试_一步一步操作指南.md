# 客户测试 — 一步一步操作指南（不用会编程）

按顺序做：**第一步 → 第二步 → 第三步**。做完一步再做下一步。  
项目文件夹里有两个可以**双击**运行的文件，用它们最简单；不会用再看下面的复制粘贴方式。

---

## 准备工作（只做一次）

1. 电脑上要有 **Node.js**（没有的话请找技术人员装好）。
2. 电脑上要有 **Docker**（用来跑 Redis；没有的话看后面「没有 Docker 怎么办」）。
3. 项目文件夹在您电脑上，例如：`C:\Users\Administrator\Desktop\openclaw-agent`。

---

## 第一步：启动 Redis（小数据库）

1. 按键盘 **Win + R**，输入 **cmd**，回车，会弹出一个黑窗口。
2. 在黑窗口里**复制下面整行**，粘贴进去，回车：

```
cd /d C:\Users\Administrator\Desktop\openclaw-agent
docker compose -f docker-compose.backend.yml up -d
```

3. 等几秒，出现 **done** 或没有报错就成功。**这个黑窗口可以关掉**。

**没有 Docker 怎么办？**  
如果提示“找不到 docker”，说明没装 Docker。请让技术人员先装 Docker，或者让技术人员在本机启动 Redis，您从第二步开始做。

---

## 第二步：启动“总控后端”（让客户端能连上）

**推荐做法（最简单）：**

1. 打开项目文件夹（例如 `C:\Users\Administrator\Desktop\openclaw-agent`）。
2. **双击** 运行 **`启动总控后端.bat`**。
3. 会弹出一个黑窗口，等大约 10～15 秒，窗口里出现：  
   **C&C backend listening on http://localhost:42888**  
   就表示成功了。
4. **这个窗口不要关**，最小化放在一边即可。

**如果不想用双击，可以用复制粘贴：**  
再按 **Win + R**，输入 **powershell**，回车，在窗口里**一次复制下面整块**，粘贴进去，回车：

```powershell
cd C:\Users\Administrator\Desktop\openclaw-agent\backend
$env:REDIS_PORT="6380"
$env:JWT_SECRET="test-secret"
$env:PORT="42888"
npm run start:dev
```

如果窗口里出现红色 **EADDRINUSE** 或“端口被占用”：用记事本打开 `启动总控后端.bat`，把里面的 **42888** 改成 **43888**，保存后再双击运行一次。

---

## 第三步：打开“网页控制台”（看大盘、设备）

**推荐做法（最简单）：**

1. 打开项目文件夹（和第二步同一个文件夹）。
2. **双击** 运行 **`启动网页控制台.bat`**。
3. 会弹出一个黑窗口，等几秒，窗口里会出现类似：  
   **Local: http://localhost:3000** 或 **http://localhost:3005**（数字以您窗口里为准）。
4. **这个窗口不要关**，最小化即可。
5. 打开**浏览器**（Chrome、Edge 都可以），在地址栏输入：
   - 窗口里是 `http://localhost:3005` 的话，就输入：**http://localhost:3005/demo.html**
   - 窗口里是 `http://localhost:3000` 的话，就输入：**http://localhost:3000/demo.html**
   - 回车。
6. 网页会跳转到“数据大盘”或演示页面，能看到数据就说明**网页测试成功**。

**如果不想用双击，可以用复制粘贴：**  
再按 **Win + R**，输入 **powershell**，回车，在新窗口里**复制下面整块**，粘贴，回车：

```powershell
cd C:\Users\Administrator\Desktop\openclaw-agent\web
npm run dev
```

然后同样看窗口里的 Local 地址，在浏览器里输入“该地址 + /demo.html”。

---

## 第四步（可选）：测试“龙虾客户端”（黑框连总控）

只有需要验证“客户电脑能不能连上总控”时才做这一步。

1. 再开一个 **PowerShell** 窗口。
2. **第一次**需要先准备配置文件：
   - 打开文件夹：`C:\Users\Administrator\Desktop\openclaw-agent\scripts\vip-build`
   - 把 **.env.vip.example** 复制一份，重命名为 **.env.vip**
   - 用记事本打开 **.env.vip**，把里面这一行改成（总控在您本机时）：  
**C_AND_C_SERVER_URL=http://127.0.0.1:42888**
     （如果第二步您把端口改成了 43888，这里就写 43888）
   - **CLIENT_DEVICE_TOKEN** 和 **MACHINE_CODE** 可以先不改，交给技术人员填。
3. 在 PowerShell 里复制下面整块，粘贴，回车：

```powershell
cd C:\Users\Administrator\Desktop\openclaw-agent
node scripts\vip-build\vip-lobster-entry.cjs
```

4. 如果成功，窗口里会出现一行绿字：  
   **🟢 龙虾节点已连接云端，等待任务下发...**  
   说明客户端连上总控了。  
   如果出现 **❌ 连接失败**，多半是第二步的后端没开，或 .env.vip 里的地址不对，请检查第二步窗口是否还在跑、地址是否和第二步的端口一致。

---

## 总结：您一共会打开几个窗口？

| 窗口       | 做什么           | 能关吗     |
|------------|------------------|------------|
| 第一个 cmd | 启动 Redis       | 做完可关   |
| 第二个 PowerShell | 启动后端（42888） | **不能关** |
| 第三个 PowerShell | 启动网页（3000/3005…） | **不能关** |
| 第四个 PowerShell | 测试 VIP 客户端（可选） | 测完可关   |
| 浏览器     | 看 http://localhost:端口/demo.html | 随时可关可开 |

---

## 常见问题

- **“找不到 npm”**：说明 Node.js 没装好或没加入 PATH，请找技术人员安装 Node.js。
- **“无法获取 /demo.html”**：您可能输错了地址。一定要用**第三步 PowerShell 窗口里显示的** Local 地址，后面加 `/demo.html`，例如 `http://localhost:3005/demo.html`。
- **“端口被占用”**：按上面第二步里说的，把 `42888` 改成 `43888` 再试；网页的端口以第三步窗口里显示的为准。

---

**说明**：本指南假设项目路径是 `C:\Users\Administrator\Desktop\openclaw-agent`。如果您的项目在别的盘或文件夹，把上面所有路径里的这一串改成您电脑上的实际路径即可。
