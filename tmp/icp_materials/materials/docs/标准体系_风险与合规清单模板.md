# ClawCommerce 风险与合规清单模板

> 总负责人要求：上线前必须完成法律合规自查表签字；本文档为风险项与合规检查的模板，由 PM/法务/技术共同维护。

---

## 1. 合规红线（未通过不得上线）

| 序号 | 检查项 | 依据 | 负责 | 状态 |
|------|--------|------|------|------|
| 1 | 个人手机号/微信的采集、存储、展示符合中国个保法及平台规则 | 个保法、平台 ToS | 后端 + PM | ☐ 通过 |
| 2 | 跨境传输（若有）符合 GDPR/PDPA 及当地要求 | GDPR/PDPA | PM/法务 | ☐ 通过 / ☐ 不适用 |
| 3 | 手机号池采购与使用可审计、无违规转售或滥用 | 内部制度 | Agent + 后端 | ☐ 通过 |
| 4 | 数据脱敏与审计日志：列表脱敏、解密即记审计 | PRD + 手册 | 后端 | ☐ 通过 |
| 5 | 平台反爬与 ToS：行为频率、指纹、账号使用符合各平台规则 | 平台 ToS | Agent + PM | ☐ 通过 |
| 6 | 无 SQL 注入、XSS、敏感信息泄露（日志/前端） | OWASP | 后端 + 前端 | ☐ 通过 |
| 7 | 密钥与权限：无 hardcode、生产密钥独立、最小权限 | 手册 | 全角色 | ☐ 通过 |

**签字**：技术负责人 ________ 日期 ________ ；PM ________ 日期 ________ 。

**对外背书**：上述条款与《标准体系_多租户数据安全与合规白皮书（供融资与大客户审计用）》一致；白皮书可直接提供给投资人、种子客户或大客户用于技术能力与合规审阅。

---

## 2. 业务与运营风险

| 序号 | 风险描述 | 等级 | 应对措施 | 负责人 |
|------|----------|------|----------|--------|
| 1 | 平台封号导致商家任务中断 | 高 | 反检测策略库、异常隔离、人工审核队列、免责与重试策略 | Agent + PM |
| 2 | 线索准确率不达标导致商家投诉 | 中 | 准确率监控、种子测试验证、产品内反馈入口 | 后端 + PM |
| 3 | 成本超支（节点/手机号/云） | 中 | 成本优化脚本、闲置释放、预算告警、月报表 | DevOps |
| 4 | 单点故障（数据库/Redis/Agent） | 高 | 高可用部署、备份与灾备、健康检查与自动恢复 | DevOps + 后端 |
| 5 | 第三方依赖（SMS、支付、平台 API）不可用 | 中 | 重试与熔断、多通道/降级方案、合同 SLA | 后端 + PM |

---

## 3. 使用说明

- 每次大版本或上线前更新本清单；新增风险项需填写等级与应对措施。
- 「合规红线」必须全部勾选通过并签字后，方可执行上线 Checklist 最终签字。
- 本文档与《项目最终交付标准手册》第 8 节「上线前最终 Checklist」配套使用。
