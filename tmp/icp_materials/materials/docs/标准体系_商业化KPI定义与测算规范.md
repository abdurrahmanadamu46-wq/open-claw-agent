# ClawCommerce 商业化 KPI 定义与测算规范

> 总负责人要求：所有「领先水平」指标必须可测算、可追溯、可汇报。本文档为唯一口径，用于种子测试、融资材料与上线验收。

---

## 1. 核心 KPI 定义表

| KPI | 目标值 | 计算公式 / 口径 | 数据来源 | 测算频率 | 负责人 |
|-----|--------|------------------|----------|----------|--------|
| **线索准确率** | ≥85% | （有效线索数 / 系统产出线索总数）× 100%。有效 = 商家在 7 日内标记为「有效」或已加微信/成单 | 后端 lead 表 + 商家标记/CRM 回传 | 每周（种子期每日） | 后端 + PM |
| **封号率** | ≤1% | （本周期内被封账号数 / 本周期参与任务的账号总数）× 100% | Agent 节点状态 + 平台封禁回调/人工登记 | 每周 | Agent |
| **内容质量评分** | ≥9/10 | PM 盲测：随机抽 20 条二创内容，5 人独立打分（1–10），取平均；单条得分 ≥9 占比 ≥80% | 人工评分表 | 每版本/每月 | PM |
| **系统可用性** | 99.99% | （总分钟数 − 不可用分钟数）/ 总分钟数 × 100%。不可用 = 核心 API 连续 1 分钟 5xx 率 >10% 或健康检查失败 | Prometheus / 健康检查探针 | 实时 + 月度报表 | DevOps |
| **UI 转化率** | ≥15% | （完成「创建任务」的 UV / 访问「配置向导或新建任务」的 UV）× 100% | 前端埋点 + 后端 campaign 创建日志 | 每周 | 前端 + PM |
| **月成本/线索** | ≤¥2 | 月度总成本（云+节点+手机号+第三方 API）/ 月度有效线索总数。有效线索同上 | 成本报表 + lead 表 | 每月 | DevOps + 后端 |
| **配置成功率** | ≥95% | （7 日内至少成功创建并运行 1 个任务的种子用户数 / 参与测试的种子用户总数）× 100% | 种子测试登记表 + campaign 表 | 种子测试结束 | PM |
| **API P95 延迟** | ≤300ms | 所有 /api/v1/* 请求的 P95 响应时间（ms） | Prometheus histogram | 实时 + 日报 | 后端 |
| **API 错误率** | ≤0.1% | （5xx 请求数 / 总请求数）× 100% | Prometheus | 实时 + 日报 | 后端 |
| **节点分配成功率** | 100% | （分配成功次数 / 后端请求分配总次数）× 100%（在有空闲节点且配额内时） | Agent 日志 + 后端调用记录 | 每周 | Agent + 后端 |
| **节点失败率** | ≤1% | （异常释放/崩溃的节点·次 / 总分配节点·次）× 100% | Agent 心跳与释放日志 | 每周 | Agent |

## 2. 测算实施要求

- **数据留痕**：所有 KPI 所用原始数据（去敏后）需可回溯至少 90 天；成本与线索数需有逐日/逐租户明细。
- **种子测试**：至少 5 名种子商家、连续 7 天；每人至少创建 1 个任务、产生不少于 10 条线索；配置成功率、线索准确率、UI 转化率必须基于该样本出具报告。
- **ROI 计算器**：产品内或独立表格，输入「线索单价、月线索量、月订阅费」等，输出「回收期、毛利率」等；公式需在用户操作手册或帮助中心写明，并每季度由 PM 校验一次。

---

## 3. 汇报与验收

- **周报**：各角色在《本周交付自查表》中填写本人负责 KPI 的当周实测值（或「未测/待上线」）。
- **上线验收**：上线前 Checklist 中「所有 KPI 达标」以本文档定义为准；不达标项须有整改计划与时间节点，经总负责人同意方可灰度。
