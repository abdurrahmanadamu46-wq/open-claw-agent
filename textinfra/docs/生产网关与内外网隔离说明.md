# 生产网关与内外网隔离说明

> 本仓（Agent + Web）与后端仓联调/生产时，由后端团队提供带 Traefik 的完整编排；此处仅记录与**安全/合规白皮书**及**本仓镜像**相关的要点，便于融资与审计说明。

---

## 1. 生产级网关（由后端团队落地）

- **Traefik** 作为边缘网关：对外 80/443，自动 HTTPS（Let's Encrypt）、安全 Header、限流等。
- **子域名隔离**：例如 `app.clawcommerce.com`（前端）、`api.clawcommerce.com`（后端 API）；Agent **不暴露在公网**。
- **CORS**：由 NestJS 精确校验 Origin（如 `https://app.clawcommerce.com`），不在网关一刀切。
- **WebSocket**：Traefik 原生支持，长连接由后端与网关共同保障。

---

## 2. 内外网流量隔离（与白皮书一致）

- **Agent 与 Backend 同属内部网络**（如 Docker 网络 `claw_net`）。
- Agent 线索回传、心跳等请求通过**内网地址**（如 `http://backend:3000`）访问 Backend，**不经过 Traefik 公网网关**。
- **效果**：
  - 内部高频心跳与线索回传**不占用公网限流**，天然豁免限流策略。
  - 公网仅暴露前端与对外 API，实现**内外网物理级隔离**，符合《多租户数据安全与合规白皮书》中的架构说明。

---

## 3. 本仓镜像在生产中的角色

- **Frontend**：由本仓 Web 构建的镜像，通过 Traefik 以 `app.clawcommerce.com` 对外。
- **Agent**：由本仓根目录 Dockerfile 构建，**无 Traefik labels**，仅接入 `claw_net`，通过 `BACKEND_INTERNAL_URL=http://backend:3000`（或团队约定路径）回传线索。
- 完整 `docker-compose.staging.yml`（含 Postgres、Redis、Mongo、Traefik 等）由后端/运维仓库维护；本仓提供 Agent 与 Web 的 Dockerfile 及 `textinfra/docker-compose.yml` 用于本地/联调。

---

## 4. 发版与域名

- 运维将 `app.clawcommerce.com`、`api.clawcommerce.com` 的 A 记录解析到云服务器公网 IP 后，执行团队约定的 `docker-compose up -d` 即可；Let's Encrypt 将自动签发证书。
- 本仓不包含生产密钥与域名配置，仅提供上述说明供联调与白皮书引用。
