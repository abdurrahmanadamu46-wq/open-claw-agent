# 后端（NestJS）登录与节点代理接口约定

前端与后端分离：**前端只请求后端**（`NEXT_PUBLIC_API_BASE_URL`），**禁止直连 Agent**。后端负责鉴权与代理 Agent 节点状态。

---

## 1. 账户登录（必接）

| 项目 | 说明 |
|------|------|
| 方法 | `POST` |
| 路径 | `/auth/login` |
| 请求体 | `{ "username": string, "password": string }`（手机号/邮箱 + 密码） |
| 成功响应 | `200`，`{ "token": string }`（JWT） |
| 失败 | `401` 或业务错误码 `40001` 等，前端会 Toast 并可选跳转登录 |

前端行为：

- 登录成功后把 `token` 写入 `localStorage.clawcommerce_token`，请求头带 `Authorization: Bearer <token>`。
- 收到 `40101`（登录过期）时清除 token 并跳转 `/login`。

---

## 2. 龙虾状态（节点监控）代理（必接）

后台需**随时能监控每个龙虾（OpenClaw 节点）状态**，由后端代理 Agent 接口，前端只调后端。

| 项目 | 说明 |
|------|------|
| 方法 | `GET` |
| 路径 | `/api/agent/nodes/status` |
| 鉴权 | 需登录，请求头 `Authorization: Bearer <token>` |
| 后端行为 | 向后端内网 Agent 发起 `GET {AGENT_INTERNAL_URL}/api/agent/nodes/status`，将响应原样或按约定格式返回前端 |
| 响应体 | 与 Agent 一致：`{ nodes: NodeStatus[], total, idle, allocated, unhealthy, at }` |

Agent 侧接口（供后端联调）：

- 本仓 Agent 服务：`GET http://localhost:38789/api/agent/nodes/status`（或 `BACKEND_INTERNAL_URL` 对应为 Agent 地址时，由后端调 Agent）。
- 后端部署时通过内网地址调 Agent，例如 `http://agent:38789/api/agent/nodes/status`。

前端「龙虾状态」页会调用 `GET /api/agent/nodes/status`（即请求后端），后端转发到 Agent 并返回，前端展示每个节点的 nodeId、state、health、campaignId、**专属网络**、**指纹浏览器**、lastHeartbeatAt 等。

---

## 3. 节点配置：专属网络 + 指纹浏览器（必接）

每个龙虾（节点）需配置**专属网络**（独立代理/出口）与**指纹浏览器**（设备指纹、反检测策略），由后端代理到 Agent 更新。

| 项目 | 说明 |
|------|------|
| 方法 | `PATCH` |
| 路径 | `/api/agent/nodes/:nodeId/config` |
| 鉴权 | 需登录 |
| 请求体 | `{ "networkConfig"?: { "proxyUrl"?: string, "region"?: string, "label"?: string }, "fingerprintProfile"?: { "profileId"?: string, "strategy"?: "random"\|"pool"\|"fixed", "poolId"?: string } }` |
| 后端行为 | 向 Agent 发起 `PATCH {AGENT_INTERNAL_URL}/internal/nodes/:nodeId/config`，body 原样转发 |
| 成功 | `200`，返回更新后的完整 NodeStatus |

- **networkConfig**：该节点专属代理（如 `http://user:pass@host:port`）、出口地域/标签。
- **fingerprintProfile**：该节点使用的指纹池/浏览器配置 ID、策略（random/pool/fixed）、poolId。

---

## 4. 前后端分离小结

| 角色 | 职责 |
|------|------|
| 前端（本仓 Web） | 登录页、商家控制台、龙虾状态页；仅请求后端 API，不配置、不直连 Agent |
| 后端（NestJS，独立仓） | 账户登录/签发 JWT、租户隔离、所有业务 API、**代理** `/api/agent/nodes/status` 到 Agent |
| Agent（本仓） | 节点池、任务执行、暴露 `GET /api/agent/nodes/status` 与 `PATCH /internal/nodes/:nodeId/config` 给后端，不对外网/前端直连 |

收到本约定后请回复：「已收到《后端_登录与节点代理接口约定》，将实现 POST /auth/login、GET /api/agent/nodes/status 代理与 PATCH /api/agent/nodes/:nodeId/config 代理。」
