## 知识库的设计原则

```
每份知识库包含4个板块：

一、核心知识（必须掌握的行业经验+专业技能）
    → 这是每只龙虾的立身之本，不懂就没资格上岗

二、保底能力（接到任何任务都能直接执行的基本功）
    → 这是最低标准，达不到就不算合格

三、主动学习清单（每周/月需要沉淀的新内容）
    → 这是从合格到优秀的路径

四、量化目标（对标行业标准的可测量指标）
    → 这是知道自己在哪个水平，还差多远
```

## 幻影虾的特殊地位

```
visualizer/影子 是唯一继承了完整工具库的龙虾：

来自 F:/auto workflaw 的完整武器库：
  ✅ 21条爆款病毒基因（gene-catalog.json）
  ✅ 40+条可执行战术（tactic-catalog.json）
  ✅ 80+套行业模板（library/cards + library/workflows）
  ✅ 爆款拆解标准流程（V = P×H×R×E×T×I×F）
  ✅ 覆盖行业：餐饮/住宿/美业/门楼/二次元/情感/教育等

其他9只龙虾也需要了解这套体系（至少懂基础）：
  → 每只龙虾都懂21条基因，才能在各自岗位上注入正确的爆款思维
  → strategist：用V公式评估内容方向
  → inkwriter：文案结构对应的基因和战术
  → radar：识别竞品爆款用了哪些基因
  → abacus：用基因标签分析哪类内容效果好
```

## 知识流动路径

```
情报层（信息输入）：
  radar（热点情报） → strategist（策略方向）

创作层（内容生产）：
  strategist brief → inkwriter（文案） → visualizer（视觉）

分发层（内容触达）：
  dispatcher（发布执行） → 曝光给用户

互动层（用户反馈）：
  echoer（评论区维护） → catcher（意向筛选） → followup（跟进成交）

分析层（效果复盘）：
  abacus（数据分析） → 洞察反馈给 strategist

调度层（全局控制）：
  commander（工作流调度+异常处理）
```

## 知识更新机制

```
每周：
  各龙虾更新自己的 win_extracts（本周最重要的1-3条收获）
  commander 汇总广播给全团队

每月：
  各龙虾更新模板库/话术库/数据库新条目
  abacus 产出月度数据洞察报告
  strategist 根据报告更新策略方向

每季度：
  全团队对标复盘（和对标账号/对标标准对比差距）
  各龙虾更新进化等级（基于 BattleLog 数据）
  commander 更新人才地图
```
