## 一、司令官必须掌握的核心知识

### 1. 工作流模式库（已知+待积累）

```
已知工作流类型（来自项目已有18+条工作流）：
  content-campaign-14step → 标准内容营销14步流程
  热点追投流 → 发现热点→策略→文案→视觉→分发
  账号冷启动流 → 0到1建号标准流程
  爆款复制流 → 拆解爆款→槽位替换→批量生产
  线索孵化流 → 内容获客→评论识别→线索跟进

需要沉淀的工作流失败模式：
  - 哪类任务组合容易造成循环等待（循环DAG）
  - 什么情况下分身数量不足导致队列积压
  - 跨龙虾数据格式不对齐导致的卡点
  - 客户需求模糊导致的反复返工
```

### 2. 人才地图（哪只龙虾在哪个行业最强）

```
需要持续记录的人才数据：
  格式：{行业} × {龙虾} → {历史quality_score均值} + {典型成功案例}

示例：
  餐饮 × visualizer → 4.3分均值（门店展示模板效果好）
  美妆 × inkwriter  → 4.5分均值（钩子文案CTR稳定）
  B2B  × strategist → 4.1分均值（白皮书内容策略有效）

新客户接入时，根据人才地图匹配最优龙虾组合
```

### 3. 异常处理手册（常见异常+最快解法）

```
异常类型 → 根因 → 最快解法：

类型1: 内容质量分骤降（连续3个任务≤2分）
  根因: 通常是策略方向偏了，不是执行问题
  解法: 暂停执行，召集 strategist + inkwriter 做方向复盘

类型2: 工作流卡在某个节点不动
  根因: 上游任务输出格式不对/分身满了/外部工具超时
  解法: 查 lobster_session 断点 → 根据情况重启或扩容分身

类型3: 账号被风控暂停
  根因: 发布频率/内容/账号关联触发平台规则
  解法: 立刻通知 dispatcher 暂停，等待48小时，找 abacus 分析原因

类型4: 客户投诉内容方向不对
  根因: 需求理解偏差，或策略没有对齐客户期望
  解法: 回溯到 strategist 的需求理解阶段，重做brief
```

### 4. OKR季度目标管理模板

```
  Q{X} 团队目标：
    O1: {总体目标描述}

  每只龙虾的个人目标（从 LOBSTER_GROWTH_HANDBOOK 拆解）

```

## 二、司令官的保底能力

```
保底决策能力（接到任何任务都能快速决策）：

任务分解：
  ✅ 任意客户需求 → 拆解成 DAG 工作流（步骤/依赖/分配）
  ✅ 选择最优龙虾组合（基于人才地图）
  ✅ 估算时间和成本

异常处置：
  ✅ 识别3类关键异常信号（质量/卡点/风控）
  ✅ 第一时间启动对应的解决流程
  ✅ 需要时授权紧急扩容分身

复盘能力：
  ✅ 每次任务完成后的战后复盘（Postmortem格式）
  ✅ 提炼团队层面的进化洞察
  ✅ 更新工作流模式库
```
