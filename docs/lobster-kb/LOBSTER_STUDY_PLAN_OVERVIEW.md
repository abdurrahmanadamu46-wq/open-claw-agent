## 📋 总体设计原则

每只龙虾的 `study_plan.json` 包含以下结构：
```
meta              → 当前等级/目标等级/对标基准
current_gaps      → 当前已知的薄弱点（诚实写，不粉饰）
weekly_routine    → 每周固定要做的4个学习动作
monthly_targets   → Q2三个月各自的重点任务
quarterly_kpis    → 季度量化考核指标
level_up_conditions → 升级到下一等级的明确条件
benchmark_accounts  → 对标学习对象
cross_domain_learning → 跨域学习计划（向哪只龙虾学什么）
```

## 📅 Q2月度节奏

### 2026-04（第一月）：打基础
```
所有龙虾的4月重点：
  ✓ 修复历史battle_log中已记录的问题
  ✓ 建立本季度的核心学习机制（每周routine稳定运转）
  ✓ 开始跨域学习的第一阶段

关键事件：
  - commander + abacus：建立月度数据复盘会机制
  - strategist + inkwriter：建立版本选择反馈机制
  - catcher + followup：制定移交包标准格式
  - echoer + inkwriter：启动用户语言回流机制
```

### 2026-05（第二月）：扩范围
```
所有龙虾的5月重点：
  ✓ 把第一月建立的机制跑稳
  ✓ 向弱势行业扩展（每只龙虾都有新行业目标）
  ✓ 开始跨域学习的第二阶段

关键事件：
  - dispatcher：建立节假日发布策略
  - visualizer：完善BGM情感地图
  - catcher：建立长周期行业评分体系
  - followup：开始验证转介绍体系
```

### 2026-06（第三月）：收尾+复盘
```
所有龙虾的6月重点：
  ✓ 整理本季度知识库（归档Q2最有价值的条目）
  ✓ 完成升级条件自评
  ✓ 为Q3计划做准备

关键事件：
  - commander组织Q2全员对标复盘会
  - 所有龙虾提交季度学习总结
  - 评估哪些龙虾满足升级条件
```

## 🔗 跨龙虾学习网络（Q2）

```
知识流向图：

radar ──情报──→ strategist ──策略方向──→ inkwriter
                    ↓                        ↓
                abacus ←──数据反馈──── dispatcher
                    ↓
            全团队洞察分发

echoer ──用户语言──→ inkwriter（文案接地气）
echoer ──意向线索──→ catcher（评分）──→ followup（跟进）
                                          ↓
                              catcher ←──成交反馈
                              
visualizer ←── inkwriter（视觉brief）
dispatcher ──格式规格──→ visualizer（提前对齐）
```

**关键CP对（Q2重点培育的协作关系）：**
- `strategist × abacus`：月度数据复盘会，数据驱动策略
- `inkwriter × echoer`：用户语言回流，文案越来越接地气
- `catcher × followup`：移交包标准化，线索不再在交接中损耗
- `radar × strategist`：情报→策略快速转化（目标：15分钟出判断）
