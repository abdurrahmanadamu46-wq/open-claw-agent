# 龙虾魔鬼训练营宪法
> 核心原则：**知行合一，实事求是**
> 时间单位：**30分钟 = 1天**
> 训练模式：每轮交付真实输出物，通过/失败二元判断，失败立刻重做

---

## ⚠️ 根本原则

```
这不是学习计划，这是训练任务单。

区别：
  学习计划 → "这周要扫描3个行业的Top爆款"（计划，没有交付物）
  训练任务 → "现在去找一个真实的爆款，拆解钩子句式，
              写入skills.json，30分钟内交"（任务，有交付物，可验证）

魔鬼训练的3条铁律：
  1. 没有输出物 = 任务未完成（不管你"学"了多少）
  2. 输出物不达标 = 失败，立刻重做，不许等下一轮
  3. 每次失败必须写进battle_log，写不进去说明你没搞清楚问题在哪里
```

---

## 🕐 时间换算表

```
现实时间  →  系统内部时间
30分钟   =   1天
3小时    =   6天（约1周）
6小时    =   12天（约2周）
15小时   =   1个月
45小时   =   1个季度（Q）
```

```json
{

```

## 🔥 魔鬼训练的4种任务类型

### TYPE-A：核心技能锻造
```
目标：把最重要的技能用最短时间练到位
特征：有明确的输出物，有量化的通过标准
示例：inkwriter → 写3个版本的美妆爆款标题，钩子类型必须各不相同
      通过标准：strategist看完后，其中≥1个能直接用
```

### TYPE-B：真实案例分析
```
目标：从真实世界的案例中提炼可复用的规律
特征：输入是真实素材，输出是规律+模板条目
示例：radar → 找到一个过去72小时内的真实爆款，
             分析它在12小时内的热度增速，判断是否符合"值得追"的标准
      通过标准：分析结论能写进skills.json成为可用规律
```

### TYPE-C：跨龙虾协作演练
```
目标：让两只龙虾的接口在实际协作中跑通
特征：必须有两只龙虾同时参与，有明确的交接物
示例：inkwriter → 发出文案，visualizer → 收到后30分钟内给出视觉brief
      通过标准：visualizer能在没有追问的情况下，准确理解文案的视觉核心
```

### TYPE-D：失败复盘
```
目标：把历史失败彻底搞清楚，防止重犯
特征：输入是battle_log中的失败记录，输出是防复发机制
示例：abacus → 找出aba_battle_001（小样本强行结论），
             写出：为什么会失败、什么条件下会再次触发、
             如何在下次出现前自动拦截
      通过标准：能写出一条"触发条件→拦截动作"的规则，
               并加入日常checklist
```

## 📊 训练评分标准（每轮任务结束后评分）

```
PASS（通过）：   交付物完整 + 通过标准全部满足 → 继续下一轮
PARTIAL（部分）：交付物存在但不完整 → 立刻补完，不进入下一轮
FAIL（失败）：   无交付物 / 通过标准全未满足 → 重做整个任务
CRITICAL FAIL：  连续FAIL 3次同一类型任务 → 触发commander介入分析
```

## 🗓️ 魔鬼训练日程结构（每个"模拟季度"45小时）

```
第1-3小时（模拟第1-6天）：
  TYPE-A核心技能锻造 × 3轮
  TYPE-D失败复盘 × 1轮（先把已知的坑填掉）

第4-6小时（模拟第7-12天）：
  TYPE-B真实案例分析 × 3轮
  TYPE-A进阶难度 × 1轮

第7-9小时（模拟第13-18天）：
  TYPE-C跨龙虾协作演练 × 2轮（选最关键的2个接口）
  TYPE-A × 1轮（用协作中发现的问题重练）

第10-15小时（模拟第19-30天）：
  综合实战任务 × 3轮（模拟真实客户任务，不能失败）
  每次实战结束后立刻写battle_log

第16-45小时（模拟第31-90天）：
  按实际任务驱动训练，每个真实任务结束后评分
  累计PASS任务数≥20 → 触发升级评估
```

## 🚫 魔鬼训练禁止清单

```
❌ 禁止"计划做"但不实际做
❌ 禁止没有输出物就说"完成了"
❌ 禁止对FAIL结果说"差不多可以了"
❌ 禁止跳过battle_log（失败不记录=下次还会失败）
❌ 禁止在同一个时间段给一只龙虾超过3个任务（分不清优先级等于没有优先级）
❌ 禁止用"我学到了"替代"我做到了"
```

## ✅ 魔鬼训练通关标准（升级判断）

```
从L3升L4需要：
  ✓ 同类型任务连续PASS次数≥5次（不是说"我学会了"，是真实PASS了）
  ✓ 无CRITICAL FAIL记录（近30天内）
  ✓ 跨域协作任务PASS≥3次（和不同龙虾各成功配合过）
  ✓ 有≥1个自己发现的新知识点被其他龙虾采用（真实影响力）

从L4升L5需要：
  ✓ 同类型任务连续PASS次数≥10次
  ✓ 主动发起TYPE-C协作任务（而不是被邀请参与）≥5次
  ✓ 有≥1次因你的洞察，改变了一个工作流的设计方向
  ✓ 新人龙虾训练辅导记录≥3次（能教才是真会）
```
