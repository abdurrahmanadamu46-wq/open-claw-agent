# 龙虾元老院 — 待补模块：回访虾与脑虫虾

与 [龙虾元老院_组织架构图与实现对应.md](./龙虾元老院_组织架构图与实现对应.md) 配套，明确两个尚未落地的云端组件的落点与实现建议。

---

## 1. 回访虾（FollowUp）

**职责**：高意向线索的即时电话/语音回访（Open Realtime API + Twilio / 语音服务）。

**数据流**：铁网虾 Catcher 或金算虾 Abacus 识别高意向线索后，触发回访虾；回访结果回写线索状态并可与 Abacus 推送 CRM 联动。

**建议落点**：

- **后端**：新建 `backend/src/follow-up/` 模块（或 `backend/src/integrations/follow-up.service.ts`）。
  - 接口：`triggerCall(leadId, tenantId, phone, metadata)`，内部调用 Twilio 或 Open Realtime API。
  - 可订阅 `lead_harvest_queue` 或由 Webhook 层在评分后触发；回访结果写 lead 表或事件流供 CRM 展示。
- **配置**：环境变量如 `TWILIO_ACCOUNT_SID`、`TWILIO_AUTH_TOKEN`、`TWILIO_CALLER_ID` 或 Realtime API 的 key；多租户可配置「是否启用回访」「回访策略」。

**依赖**：线索已落库并完成评分（金算虾）；可选与 Catcher 的「高意向」标签联动。

---

## 2. 脑虫虾（Strategist）

**职责**：对触须虾抓取的数据做分析与策略输出（Pandas + NumPy + GPT/RAG/LangChain），为吐墨虾/幻影虾提供参数（如选题、风格、目标人群）。

**数据流**：Radar 出队 → Strategist 分析 → 输出结构化策略 payload → 入队 content_forge（或直接作为 content_forge 的输入增强）。

**建议落点**（二选一或组合）：

- **方案 A**：独立队列 `strategist_queue`。
  - 新增 `backend/src/autopilot/workers/strategist.worker.ts`，消费 `radar_sniffing` 产出或上游新队列；输出写入 content_forge 的 payload（如 `strategyContext`、`targetAudience`）。
  - 需在 `autopilot.constants.ts` 增加 `STRATEGIST_QUEUE`、`STRATEGIST_DLQ`，并在 coordinator 中串入流水线。
- **方案 B**：不新队列，在 content_forge Worker 内调用「策略服务」。
  - 新建 `backend/src/agents/strategist/`（或 `backend/src/strategist/strategist.service.ts`），提供 `analyze(radarOutput): Promise<StrategyOutput>`。
  - content_forge Worker 在生成文案/视觉前先调 `strategistService.analyze(job.data)`，将结果并入 job.data 再调 golden-writer / visual-director。

**技术栈**：可先用 Node + 现有 LLM 服务（如 backend 已有 LLM module）做轻量策略；若需 Pandas/NumPy 重算，可单独 Python 服务或 Lambda，由 Node Worker 通过 HTTP 调用。

---

## 3. 验收标准（建议）

- **回访虾**：配置 Twilio 后，对单条测试线索触发回访，能接通并记录结果到 lead 或事件表；Admin/CRM 可查看「已回访」状态。
- **脑虫虾**：Radar 产出经 Strategist 后，content_forge 收到的 payload 含策略字段；或独立队列端到端跑通且 DLQ/重试正常。
