# 故障定位速查表（控制面 + AI 子服务 + 边缘）

## 0. 一键看状态

```powershell
# 全栈容器状态
npm run apps:ps

# 主链路发布回归（含 contracts）
npm run module:test:release
```

---

## 1. 控制台打不开（`http://127.0.0.1:3301`）

```powershell
npm run apps:up:web
npm run apps:logs:web
```

排查点：
- `web` 是否在 `apps:ps` 里为 `Up`
- `NEXT_PUBLIC_API_BASE_URL` 是否指向 `http://127.0.0.1:48789`

---

## 2. 登录/接口报 5xx（backend 异常）

```powershell
npm run apps:up:backend
npm run apps:logs:backend
npm run apps:test:backend
```

排查点：
- `backend` 日志中是否出现 Redis 连接失败
- `NEW_API_TOKEN`、`JWT_SECRET` 是否配置
- 若是本地环境，先跑 `module:up:control` 拉齐依赖

---

## 3. AI 不返回或跑到 fallback

```powershell
npm run apps:up:ai
npm run apps:logs:ai
npm run apps:test:ai
```

排查点：
- `ai-subservice` 健康检查是否 `healthy`
- `LOCAL_LLM_BASE_URL`（Ollama）是否可达
- 复杂图谱任务改用 `apps:up:ai-heavy`

---

## 4. RAG-Anything 多模态链路异常

```powershell
npm run apps:up:ai-heavy
npm run apps:logs:ai-heavy
```

排查点：
- `DRAGON_AI_INSTALL_HEAVY=1`
- `DRAGON_AI_RAG_ANYTHING_ENABLED=true`
- 资源不足时先降回轻量模式 `apps:up:ai`

---

## 5. 边缘节点不回传/不执行

```powershell
npm run apps:test:edge
# 需要本地跑一个模拟边缘时：
npm run apps:dev:edge
```

排查点：
- `edge_id` 是否在后台注册
- `EDGE_SHARED_SECRET` 是否一致
- `receive_dm_from_edge` 是否返回 200

---

## 6. CI 门禁失败（主干）

必过项：
- `mainline-gate-matrix / contracts`
- `mainline-gate-matrix / week3-e2e-live`

本地预跑：

```powershell
npm run contracts:validate
npm run apps:test:all
```

---

## 7. 常用恢复动作

```powershell
# 拉起控制面完整链路
npm run module:up:control

# 仅按 apps 入口拉起
npm run apps:up:all

# 清理 apps 入口服务
npm run apps:down:all
```

---

## 8. 编码/乱码快速判断

```powershell
Get-Content -Encoding UTF8 README.md | Select-Object -First 20
```

若日志乱码，优先通过统一入口运行（`scripts/apps/apps.ps1`、`scripts/modules/module.ps1`），已内置 `PYTHONUTF8=1`。
