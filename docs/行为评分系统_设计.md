# 行为评分系统（Behavior Scoring System）— 设计说明

行为评分系统让「龙虾池」形成**行为价值闭环**：执行 → 记录 → 打分 → 经验池 → 反馈到生成，实现**自动进化**。

---

## 一、目标

1. **评估行为质量**：从效果、拟人性、风险、效率多维度判断路径是否有效且拟人。
2. **给行为打分**：为每条执行记录生成综合评分。
3. **反馈到策略与生成**：高分行为入经验池，Behavior Engine 生成时优先参考/变异高分模板。
4. **自动进化**：长期迭代后，系统更稳定、自然、高效。

---

## 二、数据流

```
边缘 Behavior Runtime 执行
    → 上报行为日志 (POST /behavior/log)
    → BehaviorLoggerService.log(entry)
    → ScoringEngineService.score(entry)  多维度打分
    → BehaviorPoolService.add(scored)   高分入经验池
    → BehaviorEngineService.generatePath 时 30% 概率从 pool.sampleAndMutate() 生成
    → 新路径下发边缘执行 → 再上报 → 闭环
```

---

## 三、多维度评分

| 维度 | 含义 | 计算方式（见 scoring-engine.service.ts） |
|------|------|------------------------------------------|
| **Effectiveness** | 行为效果 | 0.1×likes + 0.3×comments + 0.2×shares + 0.4×leads，归一化 0–1 |
| **Human-likeness** | 拟人性 | 步骤 delay 均值与人类基线（1.2s±0.6s）接近程度 |
| **Risk** | 风险 | 1 − (repeated_pattern / too_fast / sync / anomaly_probability) |
| **Efficiency** | 成本效率 | leads / log(duration+1)，归一化 |

**综合分**（默认权重）：

```
total = 0.4×effectiveness + 0.3×human − 0.2×(1−risk) + 0.1×efficiency
```

权重可通过 `GET /behavior/scoring/weights` 查看，后续可扩展 `PUT /behavior/scoring/weights` 动态调节。

---

## 四、经验池与进化

- **BehaviorPoolService**：保留总分 ≥ 0.5 的记录，按分数排序，池容量 500。
- **getTemplates(limit, minScore)**：供调度或前端拉取高分模板。
- **sampleAndMutate(seed)**：随机取一条高分路径，对 step.delay 做 ±20% 变异，供 Behavior Engine 使用。
- **BehaviorEngineService.generatePath**：当 `usePool=true`（默认）且池中有模板时，约 **30% 概率** 使用 `sampleAndMutate(persona_id)` 生成路径，其余仍走规则生成。

---

## 五、API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /behavior/log | 边缘上报行为日志，返回带 score 的 ScoredBehavior |
| GET | /behavior/pool?limit=10&minScore=0.6 | 获取经验池模板与 poolSize |
| GET | /behavior/pool/top?limit=20 | 获取 Top N 带评分记录 |
| GET | /behavior/scoring/weights | 获取当前评分权重 |

---

## 六、边缘侧

- **reportBehaviorLog(baseUrl, payload)**：见 `src/agent/behavior-reporter.ts`。执行完 `runPath()` 后，将 path、duration_sec、effectiveness、step_delays_sec、node_health、risk_flags 等组装为 payload 上报。
- 后端收到后自动完成：打分 → 入池 → 日志。

---

## 七、与架构图对应

- **Behavior Logger**：BehaviorLoggerService + POST /behavior/log
- **Scoring Engine**：ScoringEngineService（多维度 + 权重）
- **Behavior Pool**：BehaviorPoolService（经验池 + 模板 + 变异）
- **Feedback Loop**：BehaviorEngineService.generatePath 内 usePool + sampleAndMutate
