# 弹性记忆模块（Elastic Memory）— 设计说明

在多 Agent 与边缘节点协同网络中，本模块的使命是：**让边缘设备拥有「连贯的灵魂」**，同时通过按设备/人设硬隔离与衰减重排，避免海量数据把数据库撑爆。

---

## 一、技术栈选型

| 组件 | 选型 | 说明 |
|------|------|------|
| **向量库** | Qdrant | Payload 级强过滤（node_id / persona_id），十万级设备极速隔离；Rust 底层，内存占用小 |
| **Embedding** | BAAI/bge-m3 | 本地部署，多语言、长文本，1024 维，开源免费 |
| **微服务** | FastAPI (Python) | 封装 LobsterMemoryEngine，供 NestJS / 9 大 Agent 调用 |

---

## 二、核心机制

1. **记忆向量化存储**：将「意图 + 上下文 + 奖励」转为自然语言描述后向量化写入 Qdrant。
2. **Payload 元数据硬隔离**：检索前按 `node_id`（及可选 `persona_id`）过滤，A 设备记忆绝不串台到 B。
3. **时间与奖励动态衰减重排**：召回后按 `final_score = similarity * exp(-0.1 * days) + 0.3 * reward` 重排，兼顾近期与高价值历史。

---

## 三、仓库实现位置

| 部分 | 路径 | 说明 |
|------|------|------|
| **Python 核心 + FastAPI** | `services/lobster-memory/` | engine.py（LobsterMemoryEngine）、main.py（REST）、requirements.txt、Dockerfile |
| **NestJS 客户端** | `backend/src/memory/` | LobsterMemoryClientService、MemoryController、memory.types.ts |
| **Behavior Engine 对接** | `backend/src/behavior/behavior-engine.service.ts` | retrieveRelevantMemories()；generatePath 入参支持 node_id / current_task，后续可据记忆偏置 steps |

---

## 四、部署指引

1. **Qdrant**：`docker run -p 6333:6333 qdrant/qdrant`（建议与 WSS Dispatch Hub 同机房）
2. **Python 记忆服务**：在 `services/lobster-memory/` 下执行 `pip install -r requirements.txt` 后 `uvicorn main:app --host 0.0.0.0 --port 8000`；或使用同目录 Dockerfile 构建镜像部署。
3. **NestJS**：设置环境变量 `LOBSTER_MEMORY_BASE_URL=http://<记忆服务>:8000`（不设则默认 localhost:8000）。后端提供 `POST /memory/store`、`POST /memory/retrieve`、`GET /memory/health` 代理到 Python。

---

## 五、与 Intent / Behavior Engine 的串联

- **写入**：边缘节点或 Agent 在完成一次「意图→上下文→结果」后，调用 `POST /memory/store`（或 Python 的 `/memory/store`），传入 node_id、intent、context_data、reward、可选 persona_id。
- **检索**：Intent Engine 或 Dispatcher 在为某节点下发任务前，调用 `POST /memory/retrieve`（或 BehaviorEngineService.retrieveRelevantMemories），传入 node_id、current_task、top_k、可选 persona_id。将返回的记忆作为「潜意识」注入 Prompt 或传入 Behavior Engine。
- **拟人度**：若回忆起「上次做 10 秒爆款短视频、5 分镜、高 reward」，本次生成的行为路径可偏置为相似结构，形成连贯且有历史延续性的动作，而非每日重置的随机行为。
