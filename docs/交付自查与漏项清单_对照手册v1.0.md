# ClawCommerce 交付自查与漏项清单（对照手册 v1.0）

> 基于《ClawCommerce 项目最终交付标准手册 v1.0》的**全项目查漏补缺**，供 PM、后端、Agent、UI/DevOps 及老板验收使用。  
> 本仓范围：**OpenClaw Agent + 前端 Web**；后端（NestJS）在独立仓库，由后端工程师自查。

**图例**：✅ 已就绪 | 🟡 部分就绪 | ❌ 待办

## 1. 通用交付原则

| 条款 | 本仓状态 | 说明 / 漏项 |
|------|----------|-------------|
| 前端 100% 调用 `src/services/` | ✅ | Web 全部经 `web/src/services/api.ts` 及 endpoints，无裸 fetch |
| ESLint + Prettier + TS 严格 | 🟡 | 有 ESLint/TS，需确认 Prettier 已接入并 PR 门禁 |
| 无 hardcode 密钥、无 console.log | 🟡 | 需全仓扫一遍并加入 lint 规则 |

| 每次 PR 更新 README/Swagger/原型/变更日志 | ❌ | 需在 CONTRIBUTING 或 PR 模板中强制要求 |
| 商业化导向 + 合规红线 | 🟡 | 脱敏/审计已有；需补「种子用户数据 + ROI」与合规自查表 |

## 8. 优先补漏建议（按角色）

- **前端（Cursor UI）**：接 WebSocket 大盘、Recharts、TanStack Table 线索表（筛选/批量/导出）、异常中心/设置/订阅三页、暗黑模式全局、Playwright E2E。
- **后端（Gemini）**：按手册自检 Mock/真实调用、RBAC、Rate Limit、AES、BullMQ、Swagger、Jest 覆盖率；与本仓 Agent 契约已对齐。
- **Agent（Cursor）**：心跳与恢复策略、50+ 行业模板与 RAG、反检测策略库文档、Playwright E2E 与周报指标。
- **DevOps（本仓）**：K8s 清单（阿里云/火山云）、GitHub Actions 全流程、Sentry/Prometheus/Grafana、成本脚本与备份方案。
