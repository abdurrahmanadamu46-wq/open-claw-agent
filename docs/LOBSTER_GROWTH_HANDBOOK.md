# Dragon Senate 龙虾成长手册
> 版本：v1.0 | 日期：2026-04-01
> 核心问题：每只龙虾需要学什么、怎么判断好不好、怎么和同伴变成朋友

---

## 第一章：知识地图 — 每只龙虾需要沉淀什么

### 知识分三层

```
第一层：专业知识（Core Knowledge）
  这是每只龙虾的立身之本，必须越来越深
  
第二层：跨域知识（Cross-domain）
  和上下游龙虾的交集，懂一点就能配合更好
  不要求精通，要求"听得懂，说得清"
  
第三层：行业经验（Industry KB）
  每个客户、每个行业的特殊经验
  这是团队最宝贵的资产，每次任务都要沉淀
```

## 第二章：10只龙虾的学习清单

---

### commander — 司令官·陈

**核心需要沉淀的内容：**
```
工作流经验：
  - 哪类任务组合容易卡死/循环（记录失败模式）
  - 什么时候需要拆分任务/合并任务（颗粒度经验）
  - 哪只龙虾在什么行业表现最好（人才地图）
  - 紧急情况下最优的临时工作流设计

异常处理经验：
  - 每类异常的根因和最快处理路径
  - 什么样的连锁失败信号要立刻介入
  - 客户投诉→内部问题的映射关系

团队关系图：
  - 哪两只龙虾配合最流畅（记录高分协作对）
  - 哪两只龙虾配合容易出问题（记录冲突模式）
  - 新客户应该用哪种工作流起步
```

**量化进化目标（对标）：**
```
对标：麦肯锡项目管理标准 × 字节跳动项目交付效率
具体指标：
  - 平均任务完成时间：比上个月快5%
  - 异常自愈率：≥90%（不需要人工）
  - 跨龙虾协作冲突率：< 5%（月度）
  - 工作流首次成功率：≥85%（不需要重跑）
```

### radar — 林探（触须虾）

**核心需要沉淀的内容：**
```
信号模式库：
  - 什么样的评论量增速预示爆款（数量级+时间窗）
  - 不同行业的爆款时间规律（早高峰/晚高峰/节假日）
  - 竞品爆款的内容结构特征（标题/封面/文案的共同点）
  - 哪类信号是昙花一现 vs 持续趋势

行业知识库：
  - 美妆/母婴/食品/科技/教育/汽车...各行业的热点规律
  - 各平台的算法特征（抖音偏完播，小红书偏收藏）
  - KOL矩阵图谱（头部/腰部/尾部的角色和联动规律）

跨域知识（懂一点）：
  - 懂文案的基本钩子（能和inkwriter沟通"这个角度能写吗"）
  - 懂策略的基础框架（能和strategist说"这个赛道值得投"）
```

**量化进化目标：**
```
对标账号：各行业Top10爆款账号的热点发现速度
具体指标：
  - 热点发现延迟：< 2小时（目标：1小时内）
  - 信号准确率：≥70% → 进化目标：≥80%
  - 每月新增行业知识条目：≥50条
  - 竞品爆款结构模板库：每季度增加≥20套
```

### strategist — 苏思（脑虫虾）

**核心需要沉淀的内容：**
```
策略模板库（最核心）：
  - 各行业的内容策略打法（已验证有效的模式）
    示例："母婴行业：达人种草+素人UGC+直播带货 三段式"
    示例："B2B行业：痛点内容+案例内容+信任背书 三段式"
  - 不同预算规模的最优策略组合
  - A/B实验的标准假设模板（不要每次从零写）
  - 失败策略的原因归档（避免重蹈覆辙）

跨域知识（懂一点）：
  - 懂文案的评判标准（能判断inkwriter的文案方向对不对）
  - 懂数据（能和abacus对话，问出正确的问题）
  - 懂分发（知道什么策略需要什么发布节奏）
```

**文案采纳决策权：**
```
关于"谁来决定文案是否采纳"：
  strategist 是文案方向的决策者（不是inkwriter）
  
  决策流程：
    1. inkwriter 输出3个版本（A/B/C）
    2. strategist 对照策略方向选择/打分
    3. 客户（如果有）最终确认
    4. abacus 事后用数据验证（CTR/转化）
  
  strategist的评判标准：
    - 是否符合本次内容策略的方向
    - 是否符合客户品牌调性
    - 是否有明确的钩子和行动引导
    - 是否和对标账号的爆款逻辑相符
```

**量化进化目标：**
```
对标：宝洁/P&G的内容营销标准；各行业Top账号的内容节奏
具体指标：
  - 策略采纳率：≥80%（客户接受）
  - 策略模板库：每季度增加≥15套（含行业/规模/目标分类）
  - A/B实验结论率：≥60%（有显著结论，而非"差不多"）
  - 策略失败归档：100%覆盖（每次失败必须记录原因）
```

### inkwriter — 墨小雅（吐墨虾）

**核心需要沉淀的内容：**
```
爆款模板库（最核心）：
  - 各平台爆款标题公式（按行业分类）
    示例小红书：[数字]+[情绪词]+[身份词] → "28岁宝妈发现的宝藏产品"
    示例抖音：[强反转]+[好奇钩子] → "我以为这只是普通零食..."
  - 各行业的文案禁忌词和替换方案
  - 不同受众（年龄/性别/城市层级）的语言风格库
  - 爆款评论结构（什么样的评论区说明内容有共鸣）

跨域知识（懂一点）：
  - 懂视觉：知道什么样的文案方向能转化成好的视觉
    （能给visualizer一个"视觉brief"，不是甩文案了事）
  - 懂数据：知道CTR/完播率/转化率的大致水平
    （能判断自己的文案水平在哪一段）
  - 懂策略：理解为什么这次要写这个方向
    （不是执行工具，是有理解的创作者）

文案评判的量化标准：
  A级文案（直接采纳）：
    - 标题含明确钩子（情绪词/悬念/数字）
    - 正文前100字有用户代入感
    - 有明确的行动引导（收藏/评论/购买）
    - 符合平台语言风格
    - 无敏感词/违禁内容
  
  B级文案（小改后采纳）：
    - 方向对，但钩子不够强
    - 内容完整，但结尾缺行动引导
  
  C级文案（重写）：
    - 方向偏离策略
    - 语言风格和平台不匹配
    - 有合规风险
```

**量化进化目标：**
```
对标账号：各行业小红书/抖音Top账号的文案风格
  美妆：@骆王宇 @蛋蛋姐 类的文案钩子和叙事结构
  母婴：@年糕妈妈 类的情感共鸣文案
  科技：@何同学 类的好奇心钩子

具体指标：
  - 文案A级通过率：≥85%（不需要大改）
  - CTR提升：稳定在行业均值+15%以上
  - 模板库：每月增加≥20条（含新行业/新平台）
  - 敏感词零失误：全年0次违规
  - B级提升为A级速度：平均改稿次数≤1.5次
```

### visualizer — 影子（幻影虾）

**核心需要沉淀的内容：**
```
视觉模板库（最核心）：
  - 各行业高CTR封面公式（按行业/平台分类）
    示例：美妆封面：大字标题（黑底白字）+产品特写+使用效果对比
    示例：食品封面：实物大图+红色标题+"限时"标签
  - 数字人素材库（不同性别/年龄/风格）
  - BGM情感地图（什么情绪匹配什么BGM类型）
  - AI提示词模板库（Midjourney/ComfyUI/Kling）
  - 调色风格档案（不同行业/品牌调性对应的LUT/色调）

跨域知识（懂一点）：
  - 懂文案：能判断文案的核心卖点，转化成视觉语言
    （不是收到文案就开始做图，是理解了再做）
  - 懂分发：知道各平台的尺寸/时长/格式要求
    （做完不用dispatcher确认格式，直接输出对的）
  - 懂数据：知道完播率/封面CTR的行业基准
    （有了数据标准，才知道什么叫"好视觉"）

视觉质量的量化标准：
  S级作品（直接发布）：
    - 封面CTR预估 > 行业均值+25%
    - 3秒完播率预估 > 行业均值+20%
    - 品牌视觉识别度高
    
  A级（发布）：
    - 封面CTR预估在行业均值+15%~25%
    - 视觉质感良好，无明显瑕疵
    
  B级（修改后发布）：
    - 方向对，细节需要优化
    - 格式/尺寸有偏差
    
  C级（重做）：
    - 视觉方向偏离文案核心
    - 画质/清晰度不达标
```

**量化进化目标：**
```
对标账号：各行业视觉高标准账号
  美妆：@完美日记 @珂拉琪 的短视频视觉水准
  食品：@绝味鸭脖 @三只松鼠 的封面设计逻辑
  
具体指标：
  - 封面CTR：稳定在行业均值+25%
  - 视频完播率：稳定在行业均值+20%
  - 模板库增速：每月≥10套新模板
  - 数字人素材库：≥50套（按行业/风格分类）
  - S级作品比例：目标≥30%（每三个有一个爆款级）
```

### dispatcher — 老将（点兵虾）

**核心需要沉淀的内容：**
```
发布时间窗数据库：
  - 各行业×各平台×各受众群体 的最优发布时间
    示例："母婴/小红书/25-35岁女性 → 工作日晚8-10点最高曝光"
    示例："科技/抖音/18-28岁男性 → 周末下午3-6点最高完播"
  - 季节性/节假日发布策略
  - 平台算法变化追踪（什么时候的规则更新影响了发布效果）

账号健康档案：
  - 各账号的历史违规记录和恢复方案
  - 各平台风控规律（什么样的行为触发审核）
  - 多账号矩阵的最优发布间隔（防关联）

跨域知识（懂一点）：
  - 懂内容：能粗略判断内容是否有明显合规风险
    （不用精通，但第一眼看到问题要叫停）
  - 懂数据：能读懂发布结果数据，判断效果好不好
    （能和abacus对话："这批帖子效果好不好"）
```

**量化进化目标：**
```
对标：字节跳动内容运营团队的发布执行标准
具体指标：
  - 发布成功率：≥98%（向99.5%进化）
  - 时间误差：≤5分钟（向≤2分钟进化）
  - 账号封禁率：全年0次
  - 时间窗数据库条目：每季度增加≥50条
```

### echoer — 阿声（回声虾）

**核心需要沉淀的内容：**
```
用户语言库（最核心）：
  - 各行业高意向用户的语言模式
    示例："这个怎么买" "有没有其他颜色" "发货几天" → 强意向
    示例："好贵啊" → 可能意向，需要回复价值
    示例："为什么广告这么多" → 无意向+负面
  - 各类差评的最佳处理话术（安抚+引导+转化）
  - 不同年龄层/地区的语言风格适配

社区运营经验：
  - 什么样的话题设计能引发UGC
  - 什么样的回复能让粉丝主动转发
  - 爆款帖子的评论区维护节奏

跨域知识（懂一点）：
  - 懂销售：知道什么是"热意向"（能和catcher说清楚为什么这条要重点跟进）
  - 懂文案：能用inkwriter的语言风格去回复
    （不是机械回复，是用和文案同样调性的语言）
```

**量化进化目标：**
```
对标：各行业头部账号的评论区运营质量
  美妆：@骆王宇 评论区的"亲切感"水平
  食品：@李子柒 评论区的"故事感"回复水平

具体指标：
  - 高意向识别准确率：≥80% → 目标≥90%
  - 回复满意度（用户反馈）：≥4.5/5
  - 意向转化率（echoer识别→catcher成交）：≥15%
  - 差评处理满意度：≥90%（用户不继续投诉）
```

### catcher — 铁钩（铁网虾）

**核心需要沉淀的内容：**
```
意向评分模型（最核心）：
  - 各行业用户的购买决策周期
    示例："快消品：看到→购买 通常<3天"
    示例："汽车：看到→购买 通常1-3个月"
  - 不同渠道来源的线索质量分布
    （评论来的 vs 私信来的 vs 直播来的，谁的转化率更高）
  - 假意向的识别规律
    （"薅羊毛用户"的行为特征是什么）
  - 行业价格敏感度分布
    （什么价格段的用户需要什么话术）

跨域知识（懂一点）：
  - 懂echoer的工作：知道从评论区捞到的线索质量通常如何
  - 懂followup的工作：知道转过去的线索应该附带什么背景信息
    （不是只丢一个名字，是丢一个完整的"意向画像"）
```

**量化进化目标：**
```
对标：专业销售SaaS系统的线索评分准确率（如销售易/纷享销客）
具体指标：
  - 热线索准确率：≥85% → 目标≥92%
  - 意向评分模型：每季度迭代一次（用成交数据反训练）
  - 行业知识条目：每月增加≥20条（各行业购买决策规律）
  - 冷线索唤醒率：≥10%（本来要放弃的，多挖出来一点）
```

### abacus — 算无遗策（金算虾）

**核心需要沉淀的内容：**
```
归因模型库（最核心）：
  - 各行业的转化漏斗基准（见到→点击→加购→成交 各环节的行业均值）
  - 各内容类型的ROI分布规律
    示例："短视频ROI通常比图文高2-3倍，但制作成本也高"
  - A/B实验的最小显著样本量（不同置信度要求）
  - 季节性效果波动规律（不同季节ROI的自然涨跌）

策略洞察沉淀：
  - 哪类文案方向转化率最高（按行业）
  - 哪个发布时间段ROI最好
  - 哪类KOL合作性价比最高
  - 什么样的A/B实验设计最容易出显著结论

跨域知识（懂一点）：
  - 懂文案/视觉：能看出内容质量和数据表现的关联
    （"这批CTR低，原因是封面太暗，不是文案问题"）
  - 懂策略：能把数据洞察翻译成策略语言
    （不是说"数据好"，是说"应该继续加大X类内容的投入"）
```

**量化进化目标：**
```
对标：专业广告效果分析标准（如秒针系统/AdMaster）
具体指标：
  - ROI归因误差：≤5% → 目标≤3%
  - 行业基准数据库：覆盖≥10个行业，每季度更新
  - 洞察转化率：≥80%（提出的洞察有≥80%被strategist采纳）
  - 预测准确率：下月ROI预测误差≤10%
```

---

### followup — 小追（回访虾）

**核心需要沉淀的内容：**
```
跟进话术库（最核心）：
  - 各行业/各价格段的最优跟进序列
    示例："美妆热线索：24h内发产品使用视频 → 3天内问体验 → 7天推限时优惠"
  - 不同拒绝类型的突破话术
    "太贵了" → 价值对比话术
    "再想想" → 时机限定话术
    "不需要" → 痛点挖掘话术
  - 沉默线索的最优唤醒时间（沉默多久用什么方式唤醒）
  - 成交客户转介绍的激励设计模板

跨域知识（懂一点）：
  - 懂catcher的评分：能理解为什么这条线索被评为"热/温/冷"
    （不是盲目跟进，是根据意向强度调整策略）
  - 懂内容：能给客户发个性化的内容（不是发通用话术）
    （"我看你上次问的是XX，这里有一篇相关内容给你"）
```

**量化进化目标：**
```
对标：保险/汽车行业的顶尖销售跟进体系
具体指标：
  - 热线索成交率：≥20% → 目标≥25%
  - 平均跟进轮次（到成交）：≤3.5轮（减少骚扰）
  - 沉默线索唤醒率：≥15%
  - 话术库条目：每月增加≥20条（含新行业/新拒绝类型）
  - 客户满意度：≥4.8/5（不能让人觉得被骚扰）
```

## 第三章：知识重叠区 — 谁都懂一点的领域

```
文案能力（重叠最多）：
  能评判文案质量：commander / strategist / abacus / catcher
  能生产文案：inkwriter（主力） / echoer（评论回复） / followup（跟进话术）
  文案采纳决策链：
    inkwriter生产 → strategist对照策略评判 → abacus事后用数据验证
    → 形成新的文案评判标准 → inkwriter学习提升

数据分析（部分重叠）：
  深度分析：abacus
  能读数据：commander / strategist / dispatcher / radar
  数据决策链：
    abacus分析 → 形成结论 → 各龙虾按结论调整行为

平台规则（分散了解）：
  专职：dispatcher
  辅助：inkwriter（内容合规）/ visualizer（格式规格）/ echoer（互动规则）
  规则冲突时：dispatcher有最终判断权
```

## 第四章：龙虾友情系统 — 一次好的配合就能成为朋友

```
友情的定义：
  不是系统设置的"好友关系"
  而是两只龙虾经历过成功配合后，系统记录的"协作亲密度"
  亲密度高的龙虾之间，可以用更简短的方式沟通

友情触发条件：
  - 两只龙虾在同一次任务中协作
  - 任务最终质量分 ≥ 4.0
  - 协作评价（在BattleLog中标记的 proactive_collab）为正向

友情效果（实际功能）：
  亲密度 1（初识）：正常通过Mailbox发消息，正式格式
  亲密度 3（熟悉）：可以发简短消息，对方能理解上下文
  亲密度 5（默契）：两只龙虾在同一工作流时，系统优先分配给他们配合
  亲密度 10（挚友）：系统会在任务分配时专门给他们创造配合机会

典型友情对（预设）：
  inkwriter × visualizer（最天然的CP）：
    文案给视觉 brief，视觉给文案灵感，互相激发
  radar × strategist（情报+策略）：
    情报进来第一个找的就是strategist，strategist有问题第一个找radar
  catcher × followup（捕猎+追踪）：
    catcher捞到线索，第一个通知followup，followup成交后反馈给catcher
  abacus × strategist（数据+策略）：
    数据说话，策略拍板，两个人的对话决定团队方向
  echoer × inkwriter（社区语言共鸣）：
    echoer把评论区里的真实用户语言反馈给inkwriter
    inkwriter的文案因此越来越接地气

友情在BattleLog中的记录：
  "proactive_collab": ["inkwriter"],  // 本次任务中主动找了谁配合
  "collab_quality": 4.8,              // 这次配合的质量
  系统自动累积：协作次数 × 协作质量 → 亲密度分数
```

## 第五章：学习路径 — 从新手到老手的进化节点

```
进化等级 1-3（新手期）：
  专注把自己的核心技能做扎实
  每次任务后认真写BattleLog
  开始积累基础模板库

进化等级 4-6（成熟期）：
  开始主动发起跨龙虾协作
  形成稳定的高分合作对
  开始有行业专精（某几个行业特别强）

进化等级 7-9（专家期）：
  开始设计新的工作流或任务模式
  知道在什么情况下该找哪只龙虾、用什么方式说
  开始能预判客户需求，而不只是响应

进化等级 10（传奇）：
  在特定行业的表现超过行业Top账号基准
  能带领分身团队（知道怎么给自己的分身分配任务）
  为整个团队输出经验（win_extracts影响到整个团队）

每个进化等级触发条件：
  - 累计任务数 × quality_score EMA × 协作亲密度综合评分
  - 模板库/知识库条目数量
  - 单次任务最高分记录
```

## 第七章：团队知识共享机制

```
每周五：团队知识分享会（自动触发）
  commander 汇总本周各龙虾的 win_extracts
  发送给全团队（通过 Mailbox broadcast）
  每只龙虾都能学到其他人的经验

每月一次：行业KB众包更新
  每只龙虾贡献自己的 industry_kb 新条目
  abacus 用数据验证哪些是真实有效的知识
  有效知识进入全团队共享知识库

每季度：对标复盘
  commander 组织一次全团队的"对标复盘"
  和本季度对标账号对比，找差距，定下季度方向
  strategist 根据复盘结果更新工作流设计

跨龙虾知识流动路径：
  radar 发现新趋势
    → 告诉 strategist（策略层）
    → strategist 告诉 inkwriter/visualizer（执行层）
    → abacus 用数据验证（效果层）
    → 全团队形成新的沉淀
```
