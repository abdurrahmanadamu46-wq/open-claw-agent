# 《ClawCommerce 项目最终交付标准手册 v1.0》

**项目代号**：ClawCommerce（基于 OpenClaw 的商业化 AI 运营 SaaS）  
**目标**：打造**行业绝对领先的可商用成品**——商家 3 分钟配置即可全自动二创运营账号、线索准确率 ≥85%、封号率 ≤1%、内容质量媲美 90%+ 人工、系统 99.99% 可用、UI 转化率 ≥15%、月成本/线索 ≤ ¥2、可直接订阅收费并融资。  
**生效日期**：立即生效  
**验收负责人**：产品经理（Gemini）+ 老板最终签字  

---

## 1. 通用交付原则（全团队必须遵守）

1. **前后端彻底分离**：前端绝不允许出现任何 fetch、业务逻辑、数据库操作，必须 100% 调用 `src/services/` 中的封装函数。
2. **代码质量门**：ESLint + Prettier + TypeScript 严格模式、单次 PR 代码审查通过率 100%、无 hardcode 密钥、无 console.log。
3. **版本控制**：所有代码必须在主分支 `main` 提交前通过 CI/CD（GitHub Actions），覆盖率不达标直接阻塞。
4. **文档同步**：每一次 PR 必须附带更新 README、Swagger、Figma 文字原型、变更日志。
5. **商业化导向**：每项功能必须证明「能让商家付费」——提供种子用户测试数据 + ROI 计算截图。
6. **合规红线**：任何涉及手机号、用户数据、平台操作的代码，必须通过 GDPR/PDPA/中国个保法/各平台反爬规则自查，未通过直接返工。

---

## 2. 前端交付标准（Cursor UI 部分）

- **架构强制**：100% 使用 React Query + WebSocket 调用 `src/services/`，无任何原生 fetch。
- **页面完整度**：
  - 配置向导页：3 步完成、行业模板一键套用、20 个对标账号批量导入、实时预览
  - 实时大盘：节点状态卡片 + Recharts 折线图 + WebSocket 实时刷新（延迟 ≤1s）
  - 线索管理页：TanStack Table + 高级筛选 + 批量操作 + 一键 CRM 推送 + Excel 导出
  - 异常中心、设置页、订阅管理页全部就绪
- **视觉与交互**：像素级对齐设计系统（Shadcn/UI + Tailwind）；暗黑/亮色模式完美切换、响应式（手机端线索查看零障碍）；loading/错误/空状态/成功提示全部美观专业。
- **性能**：首屏加载 ≤1.5s，Lighthouse 评分 ≥95 分（Performance + Accessibility + Best Practices）。
- **测试**：Playwright E2E 覆盖率 ≥90%，包含多租户隔离测试。

---

## 3. 后端交付标准（Gemini 后端工程师）

- **API 服务层**：`agent.service.ts`、`campaign.service.ts`、`lead.service.ts` 全部实现 Mock 开关 + 真实调用。
- **多租户与安全**：JWT + Tenant-ID 中间件 100% 生效；RBAC 权限体系（商家仅能看自己数据）；Rate Limiting、AES 加密（手机号/微信）、防 SQL/XSS 全部开启。
- **业务逻辑**：BullMQ 定时任务（每日触发 Agent + 线索回传）稳定运行；OpenClaw 通信层（REST + WebSocket）带重试、熔断、日志追踪；数据库 Schema（PostgreSQL + MongoDB + Redis）包含所有表 + 索引 + Migration。
- **监控与性能**：Prometheus + Grafana 暴露全部关键指标（API ≤300ms、错误率 ≤0.1%）；Swagger 文档零死角（含示例、错误码）。
- **测试**：Jest + Supertest 覆盖率 ≥85%，包含多租户隔离、并发压力测试。

---

## 4. AI Agent 核心交付标准（Cursor AI Agent 工程师）

- **节点管理**：动态分配（电脑 + 手机号池）成功率 100%；每 5 分钟心跳监控 + 自动恢复，节点失败率 ≤1%。
- **内容二创引擎**：Prompt 模板库 v1.0（50+ 行业）+ RAG 自学习；真实 Playwright 操作（发帖/点赞/评论/私信）+ 反检测策略库（指纹、UA、行为随机）；内容质量主观评分 ≥9/10（PM 盲测）。
- **线索提取引擎**：每日定时任务稳定运行；线索准确率 ≥85%（种子商家验证数据）；多渠道推送（Webhook、钉钉、企业微信、邮件）100% 成功 + 去重打分。
- **反封号能力**：成功率 ≥98%，异常账号自动隔离 + 人工审核队列。
- **测试**：Jest + Playwright E2E（包含真实平台模拟），每周运行报告必须包含：内容产出数、线索数、封号率、成本/线索。

## 5. UI/UX & DevOps 交付标准（Cursor UI/DevOps 合并角色）

**UI/UX 部分**：

- 设计系统完整（Token、组件库、暗黑模式）
- 用户旅程图（配置→启动→查看线索 ≤3 步）
- A/B 测试报告（按钮文案、布局转化率数据）
- 所有页面提供 Figma 文字原型 + 可直接运行的 Shadcn 组件代码

**DevOps 部分**：

- Docker Compose 一键启动（本地）
- Kubernetes manifests（生产就绪，**中国大陆 阿里云/火山云，可自由选择**）
- GitHub Actions CI/CD 全流程（前端/后端/Agent）通过率 100%
- 监控告警（Sentry + Prometheus + Grafana）覆盖所有服务
- 成本优化脚本（闲置节点自动释放）+ 每日成本报表
- 备份灾备方案（每日 S3 + 异地恢复测试通过）
