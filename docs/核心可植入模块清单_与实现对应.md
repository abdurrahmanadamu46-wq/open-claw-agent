# 核心可植入模块清单 — 与实现对应

本文档将「核心可植入模块清单」中的 10 个模块与 GitHub 可复用项目、本仓库实现位置、部署位置（Cloud/Edge）及接口/设计一一对应，便于按序优化与落地。

---

## 1. Agent 与技能框架

| 模块 | 来源/参考 | 部署位置 | 本仓库实现 | 接口/设计要点 |
|------|------------|----------|------------|----------------|
| **Agent 基础框架** | openclaw/openclaw | 云端 AI Agents + 边缘 Behavior Runtime | 云端：`backend/src/agents/`（黄金编剧/视觉导演/转化黑客等）；边缘：`src/agent/`（Node + Playwright）、`docker/lobster/`（OpenClaw Gateway + 技能） | 统一技能调用约定：`executeSkill(skillId, payload)`；边缘执行器见 `behavior-runtime.runPath(path, executor)` |
| **技能模板库** | awesome-openclaw-skills | Behavior Engine 调用 | 行为路径结构：`backend/src/behavior/types/behavior.types.ts`（BehaviorPath: session_id, steps[]）；可对接外部技能 JSON（id, type, steps, conditions）作为模板输入 | Behavior Engine 生成 steps 序列；未来可从技能库按 type 拉取模板再填充参数 |
| **真实用例参考库** | awesome-openclaw-usecases | Persona Engine / Intent Engine | `backend/src/behavior/persona-engine.service.ts`、`intent-engine.service.ts`；经验池：`behavior-pool.service.ts`（高分行为即用例） | 训练样本格式：`{ persona, intent, behavior_template }`；经验池 `getTemplates()`/`sampleAndMutate()` 已支持从高分行为采样 |

## 2. 管理与安全

| 模块 | 来源/参考 | 部署位置 | 本仓库实现 | 接口/设计要点 |
|------|------------|----------|------------|----------------|
| **多实例管理与资源隔离** | claworc | 边缘 + 中间件 WSS Hub | 节点生命周期：`backend/src/gateway/fleet-websocket.gateway.ts`（连接/心跳/踢人）；Redis 节点状态：`fleet:node:*`；调度侧：`backend/src/autopilot/` 队列与 Worker | 接口方向：`startNode`/`stopNode`、健康检查、容器隔离防篡改；当前为进程/容器级，可扩展为 claworc 集成 |
| **技能安全审计** | ClawGuard/PASB | Behavior Engine + 边缘 Runtime | 暂无独立模块 | 设计：在 Behavior Engine 与边缘执行前扫描 Skill JSON/payload；接口：`getSkillRisk()` 安全评分、`allowSkill()` 白名单；可新建 `backend/src/behavior/skill-security.service.ts` 或挂接 ClawGuard |

## 3. 行为智能（Behavior OS）

| 模块 | 来源/参考 | 部署位置 | 本仓库实现 | 接口/设计要点 |
|------|------------|----------|------------|----------------|
| **经验池 / 高分行为模板** | 自建 | Scoring Engine + Behavior Engine | `backend/src/behavior/behavior-pool.service.ts` | `saveHighScoreBehavior` → 由 BehaviorLogger 打分后 `pool.add(scored)`；`sampleBehavior` → `getTemplates()`/`sampleAndMutate()`；上限 500，minScore 0.5 |
| **行为评分系统** | 自建 | 四维打分，反馈 Behavior Engine | `backend/src/behavior/scoring-engine.service.ts`、`behavior-scoring.types.ts` | 四维：Effectiveness / Human-likeness / Risk / Efficiency；加权总分；`GET /behavior/scoring/weights`；打分结果入池并参与进化 |
| **边缘行为运行时** | OpenClaw + 自定义 executor | 边缘 | `src/agent/behavior-runtime.ts`（interpret/runPath）；后端解释：`behavior-runtime.service.ts` | 解析 JSON 行为路径；时间控制与噪声注入（delay 随机化）；上报：`reportBehaviorLog()` → POST /behavior/log |

## 4. 通信与动作

| 模块 | 来源/参考 | 部署位置 | 本仓库实现 | 接口/设计要点 |
|------|------------|----------|------------|----------------|
| **事件总线** | OpenClaw/ClawHub 思路 | 中间件 | `backend/src/behavior/behavior-event.bus.ts` | 事件：`high_intent_lead`（Catcher→FollowUp）、`behavior_completed`（边缘上报后）、`behavior_path_generated`（Behavior Engine 产出）；发布/订阅，供回访与跨节点协同 |
| **实时电话回访** | Open Realtime API + Twilio | 云端 FollowUp Agent | `backend/src/behavior/behavior-event.listener.ts`（监听 high_intent_lead，当前打日志） | 高意向线索触发后 10 秒内执行；输入：线索数据 + 人设话术；结果回写 CRM 与评分；待实装 Twilio/Realtime API，见 `docs/龙虾元老院_待补模块_回访虾与脑虫虾.md` |

---

## 5. 数据流小结（与流程图一致）

- **智能闭环**：Persona/Intent → Behavior Engine（可接技能库）→ Dispatcher → 边缘 Runtime → Scoring（经 Telemetry/上报）→ 经验池 → Behavior Engine 进化。
- **动作闭环**：Catcher/Echoer → Event Bus → FollowUp（Twilio）→ CRM/Scoring。
- **基础设施**：中间件（WSS / TaskState / EventBus）管理状态与 JSON 行为下发至边缘节点。
