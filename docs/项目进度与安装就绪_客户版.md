# 项目进度怎么看？—— 客户安装就绪说明

**适用：** 已有客户等待安装，需要一句话说清「能装什么 / 不能装什么 / 进度以哪份文件为准」。  
**原则：** 不依赖「感觉」，只依赖 **仓库里已存在的交付物 + 下面这张表**。

---

## 一、进度只看一个地方（推荐）

| 你想知道… | 打开这个 |
|-----------|----------|
| **本周/阶段交付了什么、差什么** | 复制 `docs/模板_本周交付自查表.md`，每周填一版，文件名可改成 `交付自查_2026-03-10.md` |
| **最终商用标准（长期目标）** | `docs/ClawCommerce_项目最终交付标准手册_v1.0.md`（偏完整产品，不是「客户第一版安装」） |

**建议：** 对客户在等安装的情况，以 **本文第二节「安装就绪矩阵」** 为准对外承诺；内部再以 **每周自查表** 推进。

- **还不能承诺的：** 「客户机器上双击即装好的 Tauri 客户端安装包」——除非你们另有已打包产物。

## 三、怎么跟客户说「进度」而不翻车

1. **先定本次安装范围**  
   - 仅演示验收 → 给 Demo 链接或内网部署 `web` + Mock。  
   - 要真绑定设备 → 必须后端 + Redis + 控制台登录 JWT + Tauri 或临时用脚本/浏览器完成 bind-ticket 流程。  

2. **每周更新一次**  
   用 `docs/模板_本周交付自查表.md` 填好「本周完成 / 未完成」，把 **安装就绪矩阵** 里某一格从 ⚠️ 改成 ✅ 时再同步客户。  

3. **文档即契约**  
   对客户承诺以 **本文矩阵 + 已提交的 md 为准**；口头说的容易对不齐。
