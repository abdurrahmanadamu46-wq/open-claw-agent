## 需要重新生产的行业列表

- 建筑行业_门楼
原因：首批旧版产物，schema/pack_type 风格与后续批次不完全一致；`radar.scoring-features.json` 与部分非核心龙虾 golden case 行业区分度偏弱，建议整包按当前 v1 风格重产。

- 企业服务_财税服务
原因：`radar.scoring-features.json` 的打分维度偏企业服务通用；部分非核心龙虾的 `expanded-golden-cases.json` 对“报税季/年报季/异常处理”等财税独有场景体现不够强，建议重做或至少补强。

- 电商出海_跨境电商
原因：整体可用，但与 `外贸工厂`、`独立站运营`、`海外仓服务` 的表达重叠度偏高，尤其 `inkwriter` 的 hooks 与问题切口还不够“纯跨境卖家”专属，建议二次重产强化子行业边界。

## 结论

- 当前知识包总体质量在可接入水平，尤其 2026-04-02 之后批量生产的 `文旅休闲` 与 `电商出海` 第二批次，结构一致性明显更好。
- 首批旧版产物与少数“企业服务/出海总类”行业，存在规则较强但评分卡和金样例略模板化的问题。
