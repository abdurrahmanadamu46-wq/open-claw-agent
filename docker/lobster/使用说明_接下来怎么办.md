# 龙虾已启动，接下来怎么用？

---

## 「需要配对」是什么意思？怎么解决？

**意思**：浏览器第一次连网关时，网关会把你当成一台「待批准设备」，需要你在**网关那侧**批准一次，之后才能用。

**操作步骤（按顺序做）：**

浏览器/Web 控制台的配对请求在 **devices（设备）** 队列，不是 nodes。请用下面命令：

1. **在浏览器里**：打开 http://localhost:18789/ ，填好 **WebSocket URL** 和 **网关令牌**，点 **「连接」**（此时会向 **devices** 队列发配对请求）。
2. **在本机 PowerShell** 查看待批准请求：
   ```powershell
   docker exec lobster-openclaw openclaw devices list
   ```
   找到状态为 **Pending** 的记录，复制其中的 **requestId**（Request Code）。
3. **批准该请求**（二选一）：
   ```powershell
   docker exec lobster-openclaw openclaw devices approve <requestId>
   ```
   或一键批准最新一条：
   ```powershell
   docker exec lobster-openclaw openclaw devices approve --latest
   ```
4. **回到浏览器**：刷新页面或再点一次 **「连接」**，即可进入控制台。

**说明**：`nodes pending` / `nodes approve` 是给**节点**（如手机/平板）用的；Web 管理界面用的是 **devices** 队列，所以要用 `devices list` 和 `devices approve`。

---

## 1. 用浏览器打开控制台（最快）

Gateway 自带 **WebChat / 控制台**，直接在浏览器访问：

- **地址**：<http://localhost:18789/>
- 若端口改过，用：`http://localhost:你设置的端口/`

**一键打开**：双击 **`打开龙虾控制台.bat`**，会自动打开默认浏览器。

首次进入若提示要配置（模型、Token 等），按页面指引在 **Settings** 里填写；我们的镜像用了 `--allow-unconfigured`，多数情况可直接开始用。

---

## 2. 在控制台里怎么用「技能」？

在 WebChat 里用自然语言和龙虾对话即可，**技能会自动被用到**：

| 你想做的事 | 怎么说 |
|-----------|--------|
| 让任意软件变成可被 AI 调用的 CLI | 「用 CLI-Anything 给 xxx 软件做个 CLI」「为这个仓库建一个 harness」 |
| 对 PDF/文档做 RAG 问答 | 「用 RAG-Anything 处理这份 PDF，然后我问你问题」「帮我对这个文档做 RAG 建库并查询」 |

- **CLI-Anything**：你说清楚「给谁做 CLI」（本地路径或 GitHub 地址），龙虾会按 SKILL 里的流程生成/优化 harness。
- **RAG-Anything**：需要文档路径；若用云端 LLM，在启动容器时传好 `OPENAI_API_KEY`（见下方「可选配置」）。

---

## 3. 用本机 CLI 连到 Docker 里的龙虾（可选）

想在终端里和龙虾对话，可以在本机装 OpenClaw CLI，并指向 Docker 里的 Gateway：

```bash
# 安装 CLI（需 Node 18+）
npm install -g openclaw@latest

# 指定 Gateway 地址（Docker 映射到本机 18789）
set OPENCLAW_GATEWAY_URL=http://127.0.0.1:18789
openclaw agent --message "用 CLI-Anything 给 GIMP 建一个 CLI"
```

Linux/macOS 用 `export OPENCLAW_GATEWAY_URL=http://127.0.0.1:18789`。

---

## 4. 健康检查

确认容器在跑：

- 浏览器打开：<http://localhost:18789/healthz>  
  若返回 200 / OK，说明 Gateway 正常。

---

## 5. 可选配置（环境变量）

在 **仓库根目录** 建 `.env`（或改 `docker/lobster/docker-compose.yml` 里的 `environment`），例如：

```env
# 端口（默认 18789）
LOBSTER_GATEWAY_PORT=18789

# RAG-Anything / 模型需要 API Key 时
OPENAI_API_KEY=sk-xxx
OPENAI_BASE_URL=https://api.openai.com/v1
```

改完后执行：

```bash
docker compose -f docker/lobster/docker-compose.yml up -d
```

---

## 6. 和「龙虾池总控」的关系

- 当前这只 Docker 龙虾是**独立**的：只在本机用，不依赖你们现有的总控后端。
- 若以后要让总控**下发任务**到这只龙虾，需要再在总控侧配置连接（例如 WebSocket 或激活码注册），把该容器当作池里的一台节点。这一步可以后续再加。

---

**总结**：先浏览器打开 <http://localhost:18789/>，在 WebChat 里直接和龙虾对话，用自然语言提到「CLI-Anything」「RAG-Anything」或「建 CLI」「文档 RAG」即可使用技能。
